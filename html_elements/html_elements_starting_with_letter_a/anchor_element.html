<a> : anchor element

The anchor element, with its href attribute is used to create hyperlinks to web pages, files, email addresses , locations in the same page, or anything else a URL can address.



Syntax

<a href="..." target="..."></a>

where href contains the URL that the hyperlink points to.



Attributes of anchor elements

These are global attributes

download:

  • Causes the browser to treat the URL as a download
  • Can be used with or without filename value
  • Without a value the browser will suggest a filename/extension, generated from various sources-
    • The Content-Disposition HTTP header
    • The final segment of the URL path
    • The media type(form of content-type header, the start os the data : URL or blob type for a blob:URL )
  • filename:
    • This suggests the filename
    • / and \ are converted to -
  • Filesystems may forbid other charactersin filenames,so browsers will adjust the suggested name if necessary.

href:

  • This is the URL that the hyperlink points to.
  • Links are not restricted to HTTP-based URLs- they can use any schema supported by browsers like:
    • Sections of a page with document fragements
    • Specific text portions with text fragements
    • Peices of media files with media fragements
    • Telephone numbers with tel: URL
    • Email address with mailto: URL
    • While web browsersmay not supportother URLschemes, websites can with registerProtcolHandler()
  • download only works for same origin policy or the blob: and data: schemes.
  • How browser treatsdownloads varies by browser,user settings,and other factors .
  • User may be prompted the download starts, or the file may be automatically saved, or it may be open automatically, either in an external application orin the browser itself.
  • If the Content-Dispositionresulting behaviour may be differnt: read mdn docs.
